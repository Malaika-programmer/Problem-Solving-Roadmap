# 30-Day Problem-Solving Roadmap

A practical 30-day roadmap to improve **logic building**, **problem-solving**, and **coding confidence** using **JavaScript**.

This roadmap is designed for learners who want to move from basic coding to real problem-solving thinking. The goal is not to memorize solutions. The goal is to understand problems, build logic, write code, debug mistakes, and explain the solution clearly.

---

## Main Goal

By the end of 30 days, you should be able to:

- Understand a coding problem clearly.
- Break a problem into smaller steps.
- Write pseudocode before coding.
- Dry run your solution with examples.
- Write brute-force solutions first.
- Improve solutions using better patterns.
- Explain time and space complexity.
- Solve beginner-to-intermediate problems without depending on AI.

## Daily Study Formula

Follow this formula every day:

```txt
20% video learning
70% hands-on coding
10% notes and revision
```

For every problem, write this structure in your notebook or GitHub repo:

```txt
1. What is given?
2. What is required?
3. Example dry run
4. Brute-force idea
5. Better approach
6. Final code
7. Time complexity
8. Space complexity
9. Mistakes I made
10. What I learned
```

## 30-Day Roadmap

| Day | Goal | Video to Watch | Hands-on Task |
|---|---|---|---|
| 1 | Understand what problem-solving means | CS50 Lecture 0: Computational Thinking | Take 5 real-life problems and convert them into input → process → output. Examples: login system, bill calculator, grade checker. |
| 2 | Learn pseudocode and dry run | CS50 pseudocode/problem-solving section | Write pseudocode for even/odd checker, largest of 3 numbers, login validation, and marks grading. |
| 3 | Strengthen conditions | freeCodeCamp JavaScript: variables and conditionals | Solve 10 condition-based problems: positive/negative, leap year, grade system, discount calculator, eligibility checker. |
| 4 | Strengthen loops | freeCodeCamp JavaScript: loops | Print patterns, multiplication table, sum of numbers, factorial using loop, and count digits. |
| 5 | Build function thinking | freeCodeCamp JavaScript: functions | Convert previous problems into reusable functions. Avoid repeated code. |
| 6 | Practice arrays basics | freeCodeCamp JavaScript: arrays | Write functions to find max, min, sum array, count even numbers, count odd numbers, and reverse array manually. |
| 7 | Weekly review | No new video | Re-solve 10 old problems without looking at previous code. Push your work to GitHub. |
| 8 | Learn searching basics | Khan Academy: intro to algorithms and linear search | Implement linear search for numbers, strings, and object search by id. |
| 9 | Learn binary search | Khan Academy: Binary Search | Implement binary search manually. Dry run with `[1, 3, 5, 7, 9, 11]`. |
| 10 | Understand sorting logic | Khan Academy: Selection Sort / Insertion Sort | Implement bubble sort and selection sort. Explain why nested loops are used. |
| 11 | Learn time complexity basics | CS50 or Khan Academy: Big O / asymptotic notation | For 10 code snippets, write expected complexity: O(1), O(n), O(n²), or O(log n). |
| 12 | Practice strings | freeCodeCamp JavaScript: strings | Solve reverse string, count vowels, palindrome check, remove spaces, and capitalize words. |
| 13 | Learn frequency counting | NeetCode: Arrays and Hashing intro | Solve count letters, count numbers, find duplicate, first repeated character, and most frequent element. |
| 14 | Weekly review and mini test | No new video | Solve 5 mixed problems in 2 hours: array max, palindrome, binary search, frequency count, and sorting. |
| 15 | Learn Two Sum pattern | NeetCode: Two Sum / HashMap explanation | Solve Two Sum using brute force first, then HashMap. Compare both approaches. |
| 16 | Learn Valid Anagram pattern | NeetCode or freeCodeCamp string algorithms | Solve Valid Anagram using sorting and using frequency object. |
| 17 | Learn two pointers | NeetCode: Two Pointers pattern | Solve palindrome with two pointers, reverse array with two pointers, and pair sum in sorted array. |
| 18 | Learn sliding window basics | NeetCode: Sliding Window intro | Solve maximum sum subarray of size k and longest substring without repeating characters. |
| 19 | Learn stack concept | NeetCode: Stack intro | Implement stack using array. Solve valid parentheses. |
| 20 | Learn queue concept | Short JavaScript queue tutorial or freeCodeCamp practice | Implement queue. Simulate ticket counter or task queue. |
| 21 | Weekly review | No new video | Re-solve Days 15–20 problems without help. Write notes about which pattern fits which problem. |
| 22 | Learn recursion basics | Khan Academy: Recursion | Write recursive factorial, countdown, sum of array, and reverse string. |
| 23 | Understand recursion dry run | Khan Academy: recursive examples | Draw call stack for factorial and Fibonacci. Then code both. |
| 24 | Learn merge sort idea | Khan Academy: Merge Sort | Implement merge two sorted arrays first, then full merge sort. |
| 25 | Learn linked list concept | NeetCode: Linked List intro | Create a Node class. Implement insert, delete, search, and print linked list. |
| 26 | Practice linked list problems | NeetCode: Reverse Linked List | Solve reverse linked list and understand cycle detection with diagrams. |
| 27 | Learn trees basics | NeetCode: Trees intro | Create a binary tree manually. Implement DFS preorder, inorder, and postorder. |
| 28 | Learn BFS / level order | NeetCode: Tree BFS | Implement level-order traversal using queue. |
| 29 | Mixed problem practice | NeetCode Roadmap practice | Solve 8 mixed problems: 2 arrays, 2 strings, 1 stack, 1 binary search, 1 recursion, and 1 tree. |
| 30 | Final assessment | No new video | Build a mini Problem Solver Notebook repo with 30 solved problems, notes, patterns, complexity, and mistakes learned. |

## Problem-Solving Method

Use this method for every coding problem:

```txt
Step 1: Understand the problem
Step 2: Identify input and output
Step 3: Write 2 to 3 examples
Step 4: Dry run manually
Step 5: Write brute-force logic
Step 6: Code the brute-force solution
Step 7: Test edge cases
Step 8: Improve the solution
Step 9: Write final code
Step 10: Explain time and space complexity
```

## Core Problem-Solving Patterns

| Problem Type | Best Pattern |
|---|---|
| Duplicate values | HashMap / Object |
| Count frequency | HashMap / Object |
| Sorted array | Binary Search / Two Pointers |
| Pair sum | Two Pointers / HashMap |
| Subarray or substring | Sliding Window |
| Matching brackets | Stack |
| Undo/redo/history logic | Stack |
| Queue-based processing | Queue |
| Repeated smaller problem | Recursion |
| Hierarchical data | Tree |
| Tree traversal | DFS / BFS |

## Important Advice

Do not copy solutions directly.

First try the problem yourself. If you are stuck, then watch the explanation. After watching, close the video and write the code yourself.

The real skill is built when you can answer this question:

> Why does this approach work?

Write that answer after every problem. That habit will build your actual logic.

---

## Final Reminder

Problem-solving is not about speed in the beginning. It is about clear thinking.

Focus on this order:

```txt
Understand → Dry Run → Brute Force → Optimize → Code → Test → Explain
```

If you follow this roadmap properly, your logic building, JavaScript confidence, and DSA foundation will become much stronger.
